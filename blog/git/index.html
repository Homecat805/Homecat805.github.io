Git 使用笔记

家猫

Git 是一个开源的分布式版本控制系统，也是内容管理系统(CMS)和工作管理系统，适用于各类项目。既可以在本地使用，也适用远程仓库。

一、本地安装

Git 可以在各种操作系统上使用，安装方式参见官方网站。对于 Ubuntu，用更新 PPA 源的方式安装，可以获得最新的稳定版本。

add-apt-repository ppa:git-core/ppa
apt update;
apt install git

二、远程仓库

常用的远程仓库有 Github 和 Gitee：

  • Github.com https://github.com;
  • Gitee.com https://gitee.com;

三、SSH 协议

SSH 为 Secure Shell 的缩写，由 IETF 的网络小组（Network Working Group）所制定；SSH 为建立在应用层基础上的安全协议，许多 Git 服务器都使用 SSH 公钥进行认证。

1、检查 SSH Key

SSH Key 存放在 ~/.ssh 目录。

ls -al ~/.ssh

2、生成 SSH Key

ssh-keygen -t rsa -C <your_email>     //生成 SSH Key

说明：生成过程中，需要回答问题，一般都是”yes“通过；生成的 Key 在 ~/.ssh 目录(/home/用户名/.ssh)，目录中的 id_rsa 是私钥 id_rsa.pub 是公钥。

3、把私钥添加到本地高速缓存

ssh-add ~/.ssh/id_rsa

4、把公钥添加到网站

cat ~/.ssh/id_rsa.pub

先将公钥复制到缓存，然后把公钥添加到网站。
以 Github 为例(Gitee基本相同)，在 github.com 注册用户成功后登录网站，找到用户头像打开菜单，依次点击 Settings > SSH and GPG Keys > New SSH Key；进入新增页面后，在 Title 栏中填入用户选定配对名称，在 Key 栏中粘帖公钥，点击 Add SSH Key ，配对完成。

5、检验配对

ssh -T git@github.com

出现 “Hi ×××! You’ve successfully authenticated, but GitHub does not provide shell access.” 信息表示配对成功。
配对成功后，就可以上传和下载代码了。
